# Design Principle & Design Pattern

## Design Principle

- 더 나은 소프트웨어 애플리케이션을 만들기 위한 high level 가이드라인을 제공하나 세세한 구현 방법을 가이드 라인으로 제공하는 것은 아니며, 어떤 프로그래밍 언어에도 적용이 가능한 디자인 원칙이다.
- 세세한 구현 단계를 알려주지는 않기 때문에 디자인 원칙을 적용하는 것은 전적으로 프로그래머에게 달려있다.
- 가장 유명한 design priciple은 SOLID가 있다.
  - the Single Responsibiity Principle (SRP)
  - Open-Closed Principle
  - Liskov Substitution Principle
  - Interface Segregation Principle
  - Dependency Inversion



## Design Pattern

- 디자인 패턴은 객체 지향 프로그래램을 구현하는데 일어나는 일반적인 문제들을 해결하기 위한 low level solution이다.
- 특정한 객체 지향 프로그래밍 문제를 해결하기 위한 구체적인 구현 방법들을 제안하는 것
- 예를 들어 하나의 클래스는 한 개의 객체만을 가지도록 구현하기 위해서 Singleton 디자인 패턴을 따르는 것을 말한다.
- 대표적인 예
  - Abstract Factory
  - Factory
  - Singleton
  - Command
